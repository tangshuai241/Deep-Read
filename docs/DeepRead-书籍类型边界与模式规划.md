# DeepRead 书籍类型边界与模式规划

> 定位：回答 DeepRead 到底适合读什么、不适合读什么，以及不同内容应切换成什么学习模式。
> 本文是产品和开发规划，不是用户营销文案。效果百分比为经验估算区间，用于判断投入优先级，不代表严格评测结果。

---

## 一、总判断

DeepRead 的核心能力不是“替人读完一本书”，而是：

```
把阅读内容转化为：说得清的理解、经得起追问的边界、接得上旧知识的联想、留得下来的 Obsidian 笔记。
```

所以它最擅长的不是所有书，而是那些需要理解、辨析、迁移、联想和长期沉淀的内容。

### 强项

- 概念理解：把抽象概念讲成自己的话。
- 边界辨析：判断一个观点什么时候成立、什么时候失效。
- 迁移应用：把书里的机制迁移到工程、投资、工作、生活。
- 知识连接：连接 Obsidian 旧笔记、概念卡、我的思考和 LLM-Wiki。
- 笔记沉淀：把对话成果整理成可复习的三段式笔记。

### 弱项

- 背诵：数字、期限、条文、定义原句，不适合只靠对话解决。
- 刷题：考试提分必须接题库、错题和题型训练。
- 精确校验：法规、标准、合同必须以原文版本为准。
- 实时更新：法律规范、软件 API、考试大纲有时效性，必须接检索或人工确认。
- 纯娱乐阅读：小说散文如果只是体验，不应强行费曼和追问。

## 三、类型细分

### 1. 概念思想型

典型内容：心理学、哲学、认知科学、社会学、经济学思想类书。

代表书：`《思考快与慢》`、认知偏误类、行为经济学类。

适配度：高。

预估效果：75%-90%。

DeepRead 能解决：

- 把抽象概念变成自己的语言。
- 识别概念之间的关系，比如 WYSIATI、确认偏误、光环效应。
- 通过苏格拉底追问发现理解漏洞。
- 把概念迁移到投资、工程、工作判断和生活经验。
- 生成适合 Obsidian 的概念笔记和双向链接。

DeepRead 不能替代：

- 原书完整阅读体验。
- 学术源流考证。
- 大规模概念图谱的人工校准。

推荐模式：`概念精读模式`。

推荐笔记形态：

- 三段式读书笔记。
- 概念卡。
- 机制链路图。
- 与旧笔记的少而准双链。

是否需要新增工具：低。当前四阶段已经基本匹配。

---

### 2. 对话疗愈型

典型内容：心理成长、人生哲学、关系、自我接纳、叙事型思想书。

代表书：`《被讨厌的勇气》`。

适配度：高。

预估效果：65%-85%。

DeepRead 能解决：

- 提取核心命题，比如课题分离、目的论、共同体感觉。
- 辨析命题前提：它为什么成立，在哪些场景会过度简化。
- 把抽象人生观落到真实关系、工作评价、家庭沟通。
- 记录“我认同什么、抵触什么、可以试什么”。

DeepRead 不能替代：

- 心理咨询或医疗建议。
- 长期行为改变。
- 情绪支持关系本身。

推荐模式：`命题辨析模式`。

推荐笔记形态：

- 命题卡：观点、前提、边界、反例。
- 个人情境卡：我在哪些场景被这个命题击中。
- 行动实验：未来一周可以试的一个小动作。

是否需要新增工具：低。主要需要调整提示词和学习契约模板。

当前四阶段适配建议：

- 不要逐段硬读。
- 不要把所有内容压成概念定义。
- 阶段 2 应偏向价值冲突、现实边界和个人场景。

---

### 3. 方法工具型

典型内容：写作、沟通、时间管理、学习方法、项目管理、效率工具。

适配度：中高。

预估效果：55%-75%。

DeepRead 能解决：

- 提炼方法框架。
- 判断方法适用条件。
- 把方法转成自己的 SOP。
- 设计行动实验和复盘问题。
- 结合工作场景生成执行清单。

DeepRead 不能替代：

- 实际执行。
- 习惯养成。
- 项目管理系统。
- 每日提醒和任务跟踪。

推荐模式：`方法转化模式`。

推荐笔记形态：

- 方法卡。
- SOP 模板。
- 适用/不适用边界表。
- 行动实验记录。

是否需要新增工具：中。

建议补充：

- 行动实验状态。
- 复盘提醒。
- 方法使用次数统计。

---

### 4. 职业技能型

典型内容：管理、投资、工程实践、业务能力、职场判断。

适配度：中高。

预估效果：60%-80%。

DeepRead 能解决：

- 抽取实践原则。
- 从案例中识别判断框架。
- 建立工作场景下的检查清单。
- 把经验沉淀到“我的思考”。
- 与工程、投资、组织管理旧笔记建立链接。

DeepRead 不能替代：

- 真实项目经验。
- 专业师傅的现场判断。
- 数据分析工具。
- 风险审批流程。

推荐模式：`方法转化模式` 或 `案例复盘模式`。

推荐笔记形态：

- 场景判断卡。
- 风险清单。
- 决策复盘。
- 案例因果链。

是否需要新增工具：中。

建议补充：

- 场景标签。
- 决策复盘模板。
- 个人经验库检索。

### 5. 考试教材型

典型内容：一级建造师、二建、法考、注会、职业资格教材。

适配度：中。

预估效果：

- 理解：45%-65%。
- 记忆：30%-50%。
- 提分：25%-45%。
- 若接入题库、错题、记忆卡，可提升到 60%-80%。

DeepRead 能解决：

- 把难理解的考点讲清楚。
- 建立章节知识结构。
- 对比易混点。
- 复盘错题背后的理解漏洞。
- 把案例题拆成答题步骤。

DeepRead 不能替代：

- 背诵。
- 刷题。
- 真题训练。
- 考点覆盖率管理。
- 考试时间压力训练。

推荐模式：`考试掌握模式`。

推荐笔记形态：

- 考点卡。
- 易混点对比表。
- 错题复盘卡。
- 记忆卡。
- 案例题答题模板。

是否需要新增工具：高。

必须补充：

- 题库。
- 错题本。
- 间隔复习。
- 考点覆盖率。
- 数字/期限/程序记忆卡。

一级建造师示例：

- 工程经济中的计算逻辑：适合 DeepRead 解释。
- 法规里的期限和程序：适合卡片和对比表。
- 管理里的网络计划：适合推导和题型训练。
- 实务案例题：适合错题复盘和答题模板。

当前四阶段适配建议：

- 不要把所有考点都做苏格拉底追问。
- 先区分“理解型考点”和“记忆型考点”。
- 理解型走对话，记忆型走卡片，题型走刷题。

## 四、推荐阅读模式

### 1. 概念精读模式

适合：概念思想型。

典型书：`《思考快与慢》`。

流程：

```
原文提取 → 核心概念地图 → 费曼输出 → 苏格拉底追问 → 强制联想 → Obsidian 笔记
```

关键指标：

- A 类核心机制是否说清。
- 是否能给出自己的例子。
- 是否能指出边界和反例。
- 是否产生 3-8 个高质量双链。

### 2. 命题辨析模式

适合：哲学、心理成长、对话体思想书。

典型书：`《被讨厌的勇气》`。

流程：

```
命题提取 → 前提辨析 → 反例/边界 → 个人情境映射 → 行动实验
```

关键指标：

- 我是否理解这个命题在反对什么。
- 我认同和不认同的部分分别是什么。
- 哪个个人场景可以试。
- 哪个场景不能乱用。

### 3. 方法转化模式

适合：工具书、职业技能书。

流程：

```
方法提取 → 适用条件 → 我的场景 → SOP → 行动实验 → 复盘
```

关键指标：

- 是否能转成自己的工作流程。
- 是否知道什么时候不用这个方法。
- 是否安排了最小实践动作。

### 4. 考试掌握模式

适合：一级建造师、职业资格教材。

流程：

```
考点拆解 → 理解/记忆分类 → 易混点对比 → 真题训练 → 错题复盘 → 间隔复习
```

关键指标：

- 考点覆盖率。
- 错题减少率。
- 易混点区分度。
- 记忆卡复习完成率。

### 5. 教材推导模式

适合：学术教材。

流程：

```
概念层级 → 原理直觉 → 推导步骤 → 典型题型 → 应用边界
```

关键指标：

- 能否复现推导。
- 能否解释公式含义。
- 能否处理典型题。

### 6. 规范检索模式

适合：法规、标准、合同。

流程：

```
版本确认 → 条文定位 → 适用条件 → 相近条文对比 → 案例/风险提示
```

关键指标：

- 是否引用到具体条文和版本。
- 是否区分强制性与建议性。
- 是否能说明适用条件。

### 7. 案例复盘模式

适合：工程案例、事故复盘、商业案例。

流程：

```
事实时间线 → 关键决策点 → 因果链 → 深层机制 → 可迁移经验
```

关键指标：

- 是否区分事实和解释。
- 是否找到关键决策点。
- 是否形成可复用的风险清单。

### 8. 文学体验模式

适合：小说、散文、纪实文学。

流程：

```
摘录 → 感受 → 人物/主题/意象 → 个人经验连接 → 长期回味
```

关键指标：

- 是否保留真实感受。
- 是否避免过度概念化。
- 是否形成可回看的摘录和感想。

### 9. 论文拆解模式

适合：论文、研究报告、行业报告。

流程：

```
问题 → 方法 → 数据/证据 → 结论 → 局限 → 对我的启发
```

关键指标：

- 是否区分结论和证据。
- 是否看见方法局限。
- 是否知道结论能用在哪、不能用在哪。

## 五、对当前项目的产品边界

### 当前版本适合继续强化的方向

优先级最高：

1. 概念精读模式。
2. 命题辨析模式。
3. 方法转化模式。

原因：

- 与现有四阶段最接近。
- 不需要立即接题库、版本库、代码执行器。
- 能最大化 Obsidian、LLM-Wiki 和个人思考库的价值。

### 当前版本不宜硬做的方向

暂不建议重度投入：

1. 完整考试系统。
2. 法规标准权威校验。
3. 技术代码自动实践。
4. 语言学习全流程。

原因：

- 它们不是单纯“对话 + 笔记”能解决。
- 需要额外系统支撑。
- 如果现在硬套四阶段，会给用户造成“我学过了”的虚假完成感。

## 七、最终结论

DeepRead 的长期方向不是做一个“读任何书都一样”的万能机器人，而是做一个本地优先、Obsidian 友好的学习系统。

它的核心边界是：

```
理解型内容：强
应用型内容：中强，但要转成方法和行动
考试型内容：中等，必须接题库和记忆系统
资料型内容：较弱，必须接检索、版本和原文定位
体验型内容：可辅助，但不能破坏阅读体验
```

如果后续要全面扩展，正确方向不是把所有书都塞进同一个四阶段，而是让 DeepRead 先判断内容类型，再选择对应的学习模式和笔记形态。
