# DeepRead Studio — Web 控制台方案

> 定位：审稿台 / 调试台 / 仪表盘，不是聊天入口

## 二、技术选型

```
后端:  FastAPI
       - 轻量，异步，自动生成 API 文档
       - 读 JSON/文件/日志 全部本地操作，无数据库
       - 一个文件 server.py，不超过 300 行

前端:  纯 HTML + 少量 vanilla JS
       - 不做 SPA，不做 React/Vue
       - 5 个页面 = 5 个 HTML 模板 (Jinja2)
       - 样式用简单的 CSS，不引入 UI 框架
       - 刷新即更新，不做 WebSocket

运行:
  python server.py
  → http://localhost:8765
```

## 三、API 设计

```
GET  /                    → Dashboard
GET  /sessions            → 会话列表
GET  /sessions/<sid>      → 单个会话详情（JSON，供前端渲染）
GET  /notes               → 笔记列表
GET  /notes/<path>        → 笔记内容 + 元数据
POST /notes/<path>/compile → 调用 compile 整理笔记
GET  /compare             → 对比页面
POST /compare/diff        → 两个笔记的差异分析
GET  /doctor              → 健康检查（调用 cli.py doctor + lark-cli）
POST /doctor/fix          → 触发修复（如重启飞书监听）
GET  /api/state           → 当前阅读状态
```

## 四、分阶段实现

### Phase 1 MVP: 会话回放 + 笔记审阅 + 对比实验

```
Phase 1a: 项目骨架（1h）
  - FastAPI + uvicorn + Jinja2
  - 基础路由 + 模板布局 + CSS
  - requirements.txt 追加 fastapi, uvicorn

Phase 1b: Sessions 回放（2h）
  - 会话列表页
  - 单会话详情：对话渲染 + 工具调用折叠
  - 数据源：state/sessions/<sid>.json

Phase 1c: Notes 审阅（1.5h）
  - 笔记列表 + 预览
  - compile 按钮（调 write_note.py compile）
  - API 用 query param 传路径（/?path=...），不用 URL path param
  - 打开文件夹按钮

Phase 1d: Compare 对比（2h）
  - 两个笔记目录选择
  - 同名文件匹配 + 并行展示
  - 指标计算：引用数/字数/内链数/frontmatter 完整性
  - 文本 diff（Python difflib）

MVP 估时: 6-8h
```

### Phase 2: 仪表盘 + 健康检查

```
  - Dashboard: 当前状态、最近会话、快速跳转
  - Doctor: 只读检查（手动点击触发，不自动跑）
  - 飞书状态显示 + 启动命令提示（不做进程管理）
```

### Phase 3: 增强功能

```
  - 飞书监听启动/停止
  - API 连通测试
  - 配置可视化编辑
```

## 五、API 路径修正

Windows 路径含反斜杠、中文、冒号，不用 URL path param，改用 query string：

```
GET  /api/notes?path=E:\Documents\知识库\读书\笔记\...
POST /api/notes/compile        body: {"path": "..."}
GET  /api/sessions
GET  /api/sessions/<sid>
POST /api/compare/diff         body: {"left": "...", "right": "..."}
GET  /api/state
GET  /api/doctor               （手动触发）
```
