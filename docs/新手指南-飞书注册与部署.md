# DeepRead 新手指南：飞书注册与部署

> 从零开始，30 分钟内完成：注册飞书机器人 → 配置权限 → 安装 DeepRead → 手机精读

---

## 概览

最终效果：

```
手机飞书 → 给 Bot 发消息 → DeepRead Agent 自动回复
例:  精读《思考快与慢》第5章 → Bot 陪你四阶段费曼精读
     查看进度 → Bot 秒回当前读书进度
     复习 → Bot 随机抽一篇旧笔记让你重新费曼输出
```

---

## 第一部分：注册飞书机器人（10 分钟）

### 1.1 打开飞书开发者后台

浏览器打开：**https://open.feishu.cn**

用你的飞书账号登录。

### 1.2 创建企业自建应用

1. 点击左侧「开发者控制台」
2. 点击「创建应用」→ 选择「企业自建应用」
3. 填写：
   - **应用名称**：`DeepRead 阅读教练`（或你喜欢的名字）
   - **应用描述**：`AI 深度阅读教练，费曼学习法 + 苏格拉底提问`
   - **图标**：随便选一个
4. 点击「创建」

### 1.3 配置权限

左侧菜单 →「权限管理」→ 搜索并添加以下权限：

| 权限 | 用途 |
|------|------|
| `im:message.p2p_msg:readonly` | 读取用户发给 Bot 的私聊消息 |
| `im:message:send_as_bot` | Bot 以自己身份回复消息 |

> 添加后点击右上角「批量开通」→ 确认开通。权限立即生效。

### 1.4 配置事件订阅

左侧菜单 →「事件订阅」→「添加事件」：

1. 搜索 `im.message.receive_v1`
2. 勾选 → 确认添加

> 不需要配置请求网址。DeepRead 通过 lark-cli 长连接收事件，不走 Webhook。

### 1.5 发布应用

1. 左侧菜单 →「版本管理与发布」
2. 点击「创建版本」→ 填写版本号 `1.0.0` → 保存
3. 点击「申请发布」→ 确认

> 企业自建应用不需要审核，发布后立即生效。

### 1.6 获取凭证

左侧菜单 →「凭证与基础信息」：

记下这两项（后面要用）：

```
App ID:     cli_xxxxxxxxxxxx
App Secret: xxxxxxxxxxxxxxxxxxxxxxxx
```

## 第二部分：安装 DeepRead（5 分钟）

### 2.1 前提

- Python 3.9+（在终端输入 `python --version` 确认）
- DeepSeek API Key（没有的话去 https://platform.deepseek.com 免费注册，充 10 元够读几十本书）
- 至少一本 EPUB 书籍

### 2.2 安装

打开终端（VS Code 里按 `Ctrl+`` `），逐行粘贴执行：

```powershell
# 1. 进入 deepread 目录
cd "E:\A工作\A、中交二航五公司\燕矶长江大桥\Claude Code 项目\deepread"

# 2. 安装依赖
pip install -r requirements.txt

# 3. 初始化配置
python init.py
```

init.py 会问你 5 个问题，一路回车用默认值即可。唯一要注意的是「EPUB 书籍目录」——填你放 EPUB 文件的路径。

### 2.3 设置 API Key

```powershell
$env:DEEPSEEK_API_KEY = "sk-你的DeepSeek密钥"
```

> 每次新开终端都要重新设置。想永久生效：在 PowerShell 里运行 `notepad $PROFILE`，在文件末尾加上这行。

## 第三部分：连接飞书（3 分钟）

### 3.1 登录新 Bot

```powershell
lark-cli auth login --domain im,event
```

这会打开浏览器，让你授权。授权完成后终端会自动继续。

> 注意：如果你之前用 lark-cli 登录过其他应用（比如 Hermes），这个命令会添加一个新身份，不会覆盖旧的。

### 3.2 测试连接

```powershell
python adapters/feishu_bot.py once "你好" --user test
```

如果正常运行，会看到：

```
后端: DeepSeek | 模型: deepseek-chat
用户: test | 会话: 20260525T...

你好！我是 DeepRead 深度阅读教练...
```

### 3.3 第一次真实消息测试（观察模式）

```powershell
python adapters/feishu_bot.py listen --max-events 1
```

终端会显示：

```
飞书事件监听启动
lark-cli 已启动，等待飞书消息...
现在用手机给 Bot 发一条消息
```

拿出手机，打开飞书，搜索你刚创建的「DeepRead 阅读教练」，发一条私聊消息：

```
查看我的读书进度
```

终端会打印出收到的原始 JSON 和回复内容。确认 `sender` 和 `text` 字段解析正确后，按 `Ctrl+C` 退出。

### 3.4 测试自动回复

```powershell
python adapters/feishu_bot.py listen --max-events 1 --reply
```

再给 Bot 发一条消息。这次手机上应该收到 Bot 的回复了。

## 第四部分：日常使用

### 4.1 启动 Bot（长期监听）

```powershell
cd "E:\A工作\A、中交二航五公司\燕矶长江大桥\Claude Code 项目\deepread"
python adapters/feishu_bot.py listen --reply
```

终端会一直运行，等待飞书消息。**不要关这个窗口。**

> 建议：把这个命令写成一个 bat 文件放桌面，双击就启动。

### 4.2 手机端的日常操作

| 你在飞书发 | Bot 做什么 |
|-----------|-----------|
| `精读《思考快与慢》第7章` | 开始四阶段费曼精读 |
| `继续` / `进入下一阶段` | 推进到下一阶段 |
| `1跳过` | 跳过当前阶段 |
| `查看进度` | 显示当前在读的书和进度 |
| `复习` | 随机抽一篇旧笔记让你重新费曼 |
| `慢思考` | 发一个值得想一天的问题，不追问 |
| `搜索 系统1` | 搜索你的 Obsidian 笔记 |

### 4.3 终端备用命令

当手机回复太长被截断，或者 Bot 暂时不可用时：

```powershell
# 终端精读（主入口）
python agent.py

# 恢复上次会话
python agent.py --resume <会话ID>

# 轻量操作（不需要 LLM）
python cli.py progress      # 查看进度
python cli.py review        # 随机复习
python cli.py think         # 慢思考
python cli.py search 关键词  # 搜索笔记
python cli.py doctor        # 健康检查
```

## 故障排查

| 现象 | 解决方法 |
|------|---------|
| `FileNotFoundError: lark-cli` | lark-cli 没安装：`npm install -g lark-cli` |
| `lark-cli 启动失败` | 先跑 `lark-cli auth login --domain im,event` |
| `another event bus already connected` | 旧进程没关：`lark-cli event stop <app_id>` |
| API 错误 | 检查 `$env:DEEPSEEK_API_KEY` 是否设置 |
| EPUB 找不到 | `python cli.py doctor` 检查路径 |
| 回复太长被截断 | 在终端用 `python agent.py --resume` 继续 |
