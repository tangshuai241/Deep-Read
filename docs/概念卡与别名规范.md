# 概念卡与别名规范

## 概念卡是什么

概念卡是 Obsidian 中 `概念（抽象概念）` 目录下的独立笔记。每张卡定义一个核心概念，是正文双链的基础节点。

DeepRead 在阶段4 compile 时，只给正文段的**概念卡**打双链。读书笔记、我的思考、Wiki 枢纽等进入"相关旧笔记 / 延伸阅读"区域。

## 文件名规范

- 用**最常用的中文名**作为文件名
- 不用英文，不用缩写（除非该领域共识用英文）
- 示例：`光环效应.md`（√） `Halo Effect.md`（×） `晕轮效应.md`（△ 应作为别名）

## 别名（aliases）规范

在 frontmatter 中配置 `aliases`，让 DeepRead 能在正文里把同义表达也链接到这张概念卡：

```yaml
---
aliases:
  - 晕轮效应
  - halo effect
  - 光环效应偏差
tags:
  - 概念卡
领域: 认知心理学
---
```

### 别名字段支持三种写法

```yaml
# 1. 多行列表（推荐）
aliases:
  - 同义词A
  - 英文名

# 2. 单行
alias: 别名

# 3. 中文键名
别名: 中文别名
```

### 别名应该放什么

| 类型 | 示例 |
|------|------|
| 同义词/翻译 | 晕轮效应 → 光环效应 |
| 英文原名 | halo effect → 光环效应 |
| 常见缩写 | WYSIATI → 眼见即为事实 |
| 逆向表述 | 系统1关闭 → 系统2启动 |

### 别名不应该放什么

- 泛泛的相关概念（应建独立卡）
- 长句子（不是概念名）
- 过于生僻的翻译

## 正文链接规则

DeepRead compile 阶段自动执行：

1. 正文 `我的理解` / `让我想到` 中，只给**概念卡**打双链
2. 概念名匹配：`光环效应` → `[[光环效应]]`
3. 别名匹配：`晕轮效应` → `[[光环效应|晕轮效应]]`
4. 每个概念每篇笔记最多正文链接一次
5. 不在已有 `[[...]]`、代码块、行内代码中重复链接
6. 不自动创建新概念卡（只会链接到已有卡）

## 质量检查

运行 `python cli.py quality <note>` 会检查：
- 正文概念双链数量是否合理（建议 1-5，>8 偏多）
- 是否有已断开的概念链接
- 延伸阅读链接是否有关系说明

运行 `python cli.py concepts aliases` 查看缺少别名的概念卡。

## 第一批建议概念（《思考快与慢》核心卡）

围绕《思考快与慢》第一部分的阅读，建议优先建立以下概念卡：

| 文件名 | aliases |
|--------|---------|
| 系统1.md | System 1, 自动系统, 快思考 |
| 系统2.md | System 2, 主动系统, 慢思考 |
| WYSIATI.md | 眼见即为事实, What You See Is All There Is |
| 确认偏误.md | 证实偏见, confirmation bias |
| 光环效应.md | 晕轮效应, halo effect |
| 情感启发式.md | 情感启发, affect heuristic |
| 替代问题.md | 问题替代, substitution |
| 可得性启发式.md | 可得性偏差, availability heuristic |
| 锚定效应.md | anchoring effect, 锚定偏差 |
| 损失厌恶.md | loss aversion, 损失规避 |

每张卡至少填入：
- `aliases`
- 一句话定义
- 关联的第一篇读书笔记链接

深度内容（核心机制、易混概念、个人例子）后续逐步补充，不追求一次写满。
