# Obsidian 笔记输出格式

## 路径规范

```
{notes_dir}/《书名》/概念名.md
```

每本书一个子目录。文件名 = 核心概念短名（如 `认知放松与真相错觉.md`）。

## 模板（obsidian-three-section）

```markdown
---
书名: 《书名》
作者: 作者名
学科:
  - 学科1
tags:
  - 读书笔记
  - 思考
章节: 5.你的直觉有可能只是错觉
---
## 📖 引用原文
> 关键原文1。注意可以包含**加粗**强调

> 关键原文2。每段引用单独一个 > 块

## 💭 我的理解
- 第一层理解要点
    - 嵌套细节用 tab 缩进展开
    - 可以继续展开
- 第二层理解要点
    - 可以引用内部概念 [[其他笔记]]
    - 可以嵌入图片

## 🔗 让我想到
- 第一层联想
    - 具体展开的细节用 tab 缩进
    - 可以是很长的生活场景描述
- 第二层联想 [[旧笔记名]]
    - 嵌套展开

## ❓ 待探索
- 【理解缺口】未分清的概念边界、机制关系或推理问题
- 【应用缺口】知道道理但还不知道如何在真实场景识别/使用的问题
- 【连接缺口】需要和旧笔记、后续章节、个人经验继续打通的问题
```

## 格式要点

- 章节号：`5.你的直觉有可能只是错觉`（数字.标题，无空格），**不加**"第X章"前缀
- 章节号不能只写数字；整章阅读也必须写成 `8.我们究竟是如何作出判断的？`
- `📖 引用原文`：每段原文独立 `>` 块，可含 `**加粗**` 强调
- `💭 我的理解` 和 `🔗 让我想到`：用 **tab 缩进**（不是空格）
- `💭 我的理解` 和 `🔗 让我想到` 的正文内链只链接概念卡，例如 `[[光环效应]]`、`[[系统1]]`、`[[情感启发式]]`
- 我的思考、其他书籍笔记、同书旧章节不放进正文内链，统一放入 `### 相关旧笔记 / 延伸阅读`
- 一级 bullet 的第一句可加粗作为回看锚点；优先加粗完整判断句，不要把每个词都加粗
- **不加 `date` 字段**
- tags 保持 `- 读书笔记` 和 `- 思考`，可额外追加概念标签
- 概念标签可选，来自 LLM-Wiki SCHEMA 五类
- `📖 引用原文` **只放 2-5 句关键原文**，不放整节全文
- `❓ 待探索` 为可选段落

## 引用选择标准（必须同时满足）

1. 必须是作者原文，不加工不概括
2. 能表达作者的核心观点（不是辅助数据或实验细节）
3. 具有总结性——读者不看正文只看引用就能大概知道这节在说什么
4. 覆盖本节的主要角度，不只挑一个侧面

## 渐进式写入

不是对话结束一次性生成，而是每阶段结束立即写入：

| 阶段 | 写入动作 | 脚本命令 |
|------|---------|---------|
| 阶段1结束 | 创建笔记草稿 | `python write_note.py create --book ... --concept ...` |
| 阶段2结束 | 覆盖更新 `💭 我的理解` | `python write_note.py update --path ... --section 我的理解 --content "..."` |
| 阶段3中 | 每有1条联想追加 | `python write_note.py append --path ... --section 让我想到 --content "..."` |
| 阶段4 | 补全 frontmatter + `❓ 待探索` | `python write_note.py finalize --path ... --tags "..." --explore "..."` |
| 阶段4最后 | 整篇笔记编译整理 | `python write_note.py compile --path ...` |

## 最终编译要求

阶段4必须在 `finalize` 后调用 `compile`，把对话过程中的粗糙片段整理成 Obsidian 成品笔记。

- 不允许保留字面量 `\n`、`\n\n`
- `引用原文` 必须是多个独立 quote block
- `我的理解` 应该有清楚层级，可用 `###` 小标题、列表、表格；优先采用“一级 bullet 核心判断 + tab 缩进解释”
- `让我想到` 应该拆成可复用段落，优先加入少而准的概念卡链接
- 正文链接规则：概念卡进正文，旧读书笔记/我的思考/Wiki 枢纽进延伸阅读；不要把正文变成知识库索引
- `让我想到` 必须保留用户在对话中提出的个人化例子、跨领域例子和现实场景，不允许只保留自动生成的“相关旧笔记 / 延伸阅读”
- 每篇成品笔记默认保留 3-8 个高质量链接；正文只放强相关链接，弱相关/启发性链接放入 `### 相关旧笔记 / 延伸阅读`
- 链接类型优先级：核心机制链接 > 具体表现链接 > 应对方法链接 > Wiki 枢纽链接 > 个人经验链接 > 延伸阅读链接
- 每个延伸链接后应保留一句关系说明，避免为了链接而链接
- 所有 `[[双链]]` 必须优先使用 Obsidian 中真实存在的文件名；未确认存在的概念名只能作为普通文本，不能自动造断链
- `待探索` 每个问题单独一条 bullet，且必须使用 `【理解缺口】`、`【应用缺口】`、`【连接缺口】` 三类之一
- `待探索` 默认保留 1-3 条；不要为了显得高级而生成抽象问题
- 好的 `待探索` 应该让读者下次看到时愿意继续想，能帮助复习、应用或连接旧知识
- 不要把对用户的聊天语气、阶段提示、"已写入" 等过程文本写进笔记

## 对话到笔记的沉淀规则

用户在对话中说出的高价值个人理解，应按用途拆分：

- 作者机制、概念定义、推理链 → 写入 `我的理解`
- 法律、工程、投资、工作、生活等用户自己的例子 → 写入 `让我想到`
- 尚未解决的问题、可继续追问的问题 → 写入 `待探索`，并按三类认知缺口分类：
  - `【理解缺口】`：当前还没真正分清的概念、机制、边界
  - `【应用缺口】`：知道道理但还不会在工程/投资/工作/生活中识别或使用
  - `【连接缺口】`：和旧笔记、后续章节、个人经验之间还有关系没打通

如果用户的例子能独立复用，必须保留场景细节。不要把“第一次去工地看到进度至上/现场杂乱会建立错误基准线”压缩成“首次输入会影响基准线”这种抽象句。

## 章节字段语义

frontmatter 的 `章节` 表示大章，不表示小节。例如阅读第7章第三节时仍写：

```yaml
章节: 7.字母"B"与数字"13"
```

小节名（如 `光环效应与群体的智慧`）放在文件名、正文标题、状态字段或概念名中，不覆盖 `章节`。

## 模板切换

config.yaml 中 `note.template` 决定模板。阶段1只支持 `obsidian-three-section`。
