# 阅读模式

DeepRead 支持 8 种阅读模式，每种模式影响追问方向、阶段通过标准和笔记结构。

## 模式判断规则

- 用户开始读一本新书时，先用 `reading_mode suggest` 判断适合的模式。
- 判断明确时直接进入，不多问。
- 不确定时只问一个选择问题。
- 用户说"考试模式""工具书模式""概念精读模式"时用户优先，立刻切换。
- 切换模式后更新 `learning_contract`，并简短说明"接下来我会按XX方式带你读"。

## 8 种模式

### 概念精读 (concept_deep_read)
- 适用：概念密集型书籍（如《思考快与慢》）
- 流程：四阶段费曼→苏格拉底→联想→收尾
- 重点：A 类核心概念必须覆盖
- 笔记：三段式（引用原文/我的理解/让我想到/待探索）
- Trial: 可用

### 命题辨析 (proposition_dialogue)
- 适用：观点型、哲学类书籍（如《被讨厌的勇气》）
- 流程：逐命题追问作者主张、前提、边界
- 重点：必须有个人立场或反例
- 笔记：核心命题/作者论证/我的立场反例/待探索
- Trial: 可用

### 方法转化 (method_conversion)
- 适用：工具书/方法论
- 流程：提取方法→验证前提→改造→归档
- 重点：阶段 3 必须产生至少 1 个行动实验
- 笔记：方法提取/前提条件/我的场景/改造方案/行动清单
- Trial: 可用

### 考试掌握 (exam_mastery)
- 适用：教材/资格证考试（如一级建造师）
- 流程：高频考点→理解→记忆→自测
- 重点：阶段推进以能否答题为主
- 笔记：考点梳理/核心理解/易错对比/自测题/待复习
- Trial: 可用

### 教材推导 (textbook_derivation)
- 适用：理论教材/数学/物理
- 笔记：推导链还原/关键步理解/边界条件/课后验证
- Trial: 不可用

### 规范检索 (standard_lookup)
- 适用：工程规范/标准
- 笔记：条文定位/条文理解/工程场景/边界说明
- Trial: 不可用

### 案例复盘 (case_review)
- 适用：商业/工程案例
- 笔记：案例事实/决策链还原/替代方案/教训提炼/可复用原则
- Trial: 不可用

### 文学体验 (literature_experience)
- 适用：小说/散文/传记
- 笔记：情境还原/人物主题理解/我的共鸣/延伸联想
- Trial: 不可用

## 契约字段

`learning_contract.json` 中：
```json
{
  "profile": "personal",
  "scope": {
    "book_type": "概念思想型",
    "reading_mode": "concept_deep_read",
    "mode_reason": "默认概念精读"
  }
}
```
